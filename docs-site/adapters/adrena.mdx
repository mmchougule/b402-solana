## Why Adrena (not Drift, not Jupiter Perps)

From PRD-16 §2:

- **Jupiter Perps** uses an async request-queue + keeper model. Doesn't
  fit the v1 synchronous adapter ABI. The path forward is PRD-14
  two-phase claim notes; PRD-14 itself is deferred until the first
  async adapter is prioritized.
- **Drift v2** experienced a $285M exploit on April 1, 2026 (DPRK
  durable-nonce admin takeover, fake CVT collateral). Tether-led
  rescue closed April 16; full reboot is pending fresh audits. Program
  ID will likely change post-reboot, so integrating against pre-reboot
  bytecode is wasted work.
- **Adrena** is operational, audited (Ottersec — one shipped, second in
  progress per Adrena docs), synchronous (no keeper), and has a public
  IDL at `AdrenaFoundation/adrena-abi`. The synchronous shape matches
  the v1 adapter ABI; the per-position PDA fits PRD-13 shadow PDAs.

Adrena's program is GPL-3.0; b402's adapter does not link to or
redistribute Adrena's source — it CPIs into the deployed Adrena program
by program ID. Runtime CPI is not a derivative work under GPL-3.0;
b402's licensing is unaffected.

## Operations (planned)

Adrena does not expose generic `open_position` / `close_position`
instructions — long and short are separate ixs, and partial close is
`close_position_*` with `params.percentage`. The handler dispatches to
the right pair via the `side` byte.

| Action variant | Adrena ix | Inputs | Outputs | Notes |
|---|---|---|---|---|
| `OpenPosition { side: 0 }` | `open_position_long` | `[(USDC, collateral)]` | `[]` (delta-zero) | Collateral in; position created |
| `OpenPosition { side: 1 }` | `open_position_short` | `[(USDC, collateral)]` | `[]` (delta-zero) | |
| `IncreasePosition` | `increase_position_long/short` | `[(USDC, additional)]` | `[]` (delta-zero) | Mutates existing position |
| `DecreasePosition` | `close_position_long/short` w/ `percentage < 1e10` | `[]` | `[(USDC, settlement_min)]` | Partial close |
| `ClosePosition` | `close_position_long/short` w/ `percentage = 1e10` | `[]` | `[(USDC, settlement_min)]` | Full close |
| `LiquidateGuardrail` | `close_position_*` (NOT `liquidate_*`) | `[]` | `[(USDC, settlement_min)]` | Pre-liquidation user-side close, avoids paying the liquidator fee |

`OpenPosition` and `IncreasePosition` are **delta-zero ops** — no
tokens come back. This is the canonical use case for the
`allows_delta_zero` registry flag introduced in PRD-04 §7 and specced
fully in [PRD-15](https://github.com/b402-ai/b402-solana/blob/main/docs/prds/PRD-15-delta-zero-deadline.md).

## Action payload

Verbatim from [`programs/b402-adrena-adapter/src/lib.rs`](https://github.com/b402-ai/b402-solana/blob/main/programs/b402-adrena-adapter/src/lib.rs):

```rust
pub enum AdrenaAction {
    OpenPosition {
        custody: Pubkey,
        side: u8,                  // 0 = long, 1 = short
        collateral_amount: u64,
        size_usd: u64,
        slippage_bps: u16,
    },
    IncreasePosition {
        position: Pubkey,
        collateral_delta: u64,
        size_delta_usd: u64,
        slippage_bps: u16,
    },
    DecreasePosition {
        position: Pubkey,
        collateral_delta: u64,
        size_delta_usd: u64,
        slippage_bps: u16,
    },
    ClosePosition { position: Pubkey, slippage_bps: u16 },
    LiquidateGuardrail { position: Pubkey, max_health_factor_bps: u16 },
}
```

## Shadow PDA

Per PRD-13, each shielded user's Adrena position is keyed off a unique
shadow PDA:

```
shadow_pda = PDA([
    "b402-shadow",
    ADRENA_PROGRAM_ID,
    b"adrena:position:v1",
    viewing_key_commitment,
])
```

The shadow PDA *is* the Adrena Position PDA — Adrena's program owns it.
This gives each shielded user their own independent Adrena position,
correctly isolating liquidation risk and L2 linkability.

## Validation plan

Mainnet-fork local validator with Adrena programs cloned, mirroring
[`examples/swap-e2e-jupiter.ts`](https://github.com/b402-ai/b402-solana/blob/main/examples/swap-e2e-jupiter.ts):

1. Boot test validator with Adrena programs cloned from mainnet.
2. Open position with $1k USDC collateral, 5x leverage long SOL.
3. Wait one slot, increase by $500.
4. Decrease by half.
5. Close.
6. Verify final settlement equals expected (within slippage).

Until the v2 ABI lands and this test runs green, Adrena remains "PRD
only."
