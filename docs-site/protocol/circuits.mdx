Both circuits are written in Circom 2.2 against `iden3/circomlib v2.4.0`.
Sources in `circuits/`:

- [`circuits/transact.circom`](https://github.com/b402-ai/b402-solana/blob/main/circuits/transact.circom)
- [`circuits/adapt.circom`](https://github.com/b402-ai/b402-solana/blob/main/circuits/adapt.circom)
- `circuits/lib/` — sub-templates: `commitment`, `nullifier`,
  `spending_pub`, `merkle_verify`, `range_check`.

R1CS info files in `circuits/build/<name>.r1cs.info.txt` are generated by
snarkjs at compile time and are the source of truth for constraint
counts.

### Public inputs (18)

The order is binding — must match `TransactPublicInputs` in
[`programs/b402-pool/src/instructions/shield.rs`](https://github.com/b402-ai/b402-solana/tree/main/programs/b402-pool/src/instructions)

| 17 | `recipientBindTag` | Domain tag `"b402/v1/recipient-bind"` |

### In-circuit constraints

Per-input note (i ∈ {0, 1}):

1. Spending pub recompute: `spendingPub_i = Poseidon_2(spendKeyPubTag,
   spendingPriv_i)`. Proves ownership without revealing the key.
2. Commitment recompute: `commitmentIn_i = Poseidon_5(commitTag,
   tokenMint_i, value_i, random_i, spendingPub_i)`.
3. Merkle path verification: 26 × Poseidon_3 hashes; `mkRoot.computedRoot
   == merkleRoot` enforced selector-guarded by `(1 - inIsDummy_i)`.
4. Nullifier recompute: `nullifier_i = Poseidon_3(nullTag, spendingPriv_i,
   leafIndex_i)`, gated by `(1 - inIsDummy_i)`.
5. Range check: `value_i < 2^64`.

Per-output note (j ∈ {0, 1}):

1. `commitmentOut_j = Poseidon_5(commitTag, tokenMint, value_j, random_j,
   spendingPub_j)`, gated by `(1 - outIsDummy_j)`.
2. Range check: `value_j < 2^64`.

Global:

1. **Token consistency.** All non-dummy input and output notes carry the
   same `tokenMint` if `publicAmountIn > 0` or `publicAmountOut > 0`.
2. **Balance conservation:** `Σ inValue (non-dummy) + publicAmountIn ==
   Σ outValue (non-dummy) + publicAmountOut + relayerFee`. Enforced as
   an `Fr` equality (4 × 2^64 ≪ p, no wrap).
3. **Public-amount exclusivity:** `publicAmountIn * publicAmountOut == 0`.
4. **Relayer fee binding:** `relayerFeeBind == Poseidon_3(feeBindTag,
   recipient_Fr, relayerFee)`.
5. **Recipient binding:** `recipientBind == Poseidon_3(recipientBindTag,
   recipientOwnerLow, recipientOwnerHigh)`. Both are 128-bit halves of the
   destination ATA's owner pubkey, range-checked to 128 bits.
6. **Nullifier ordering:** `nullifier[0] < nullifier[1]` when both are
   non-dummy (canonicalizes duplicate-nullifier detection on chain).

## `adapt` circuit (composability)

From `circuits/build/adapt.r1cs.info.txt`:

```
Curve:           bn-128
Wires:           17,654
Constraints:     17,582
Private inputs:  128
Public inputs:   23
Labels:          61,450
```

### Public inputs (23)

The first 18 are byte-identical to `transact` for prover-side code reuse.
5 are adapt-specific:

| # | Signal | Meaning |
|---|---|---|
| 0–17 | (transact prefix) | Same as transact |
| 18 | `adapterId` | `keccak256(adapter_program_id)` reduced mod p |
| 19 | `actionHash` | `Poseidon_3(adaptBindTag, keccak(action_payload)_Fr, expectedOutMint_Fr)` |
| 20 | `expectedOutValue` | Pool's minimum OUT-mint delta after CPI |
| 21 | `expectedOutMint` | OUT mint, reduced mod p |
| 22 | `adaptBindTag` | Domain tag `"b402/v1/adapt-bind"` |

### Differences vs transact

From [`circuits/adapt.circom`](https://github.com/b402-ai/b402-solana/blob/main/circuits/adapt.circom):

1. **Two mint bindings.** Input notes bind to `publicTokenMint` (IN
   mint); output notes bind to `expectedOutMint` (OUT mint), enforced
   by feeding `expectedOutMint` into each `commitOut[k].tokenMint`.
2. **Balance constraint changes.** `inSum === publicAmountIn +
   relayerFee` (input notes consumed by the adapter call + fee). `outSum
   === expectedOutValue` (output notes' total value reflects the floor
   the adapter must deliver).
3. **`publicAmountOut` forced to 0.** Adapt has no public unshield path.
4. **Action hash binding.** `actionHash === Poseidon_3(adaptBindTag,
   actionPayloadKeccakFr, expectedOutMint)`. Pool on-chain recomputes
   `keccak256(action_payload)` mod p and verifies; closes the
   relayer-tampered-payload attack.
5. **`adapterId` passthrough.** The pool program enforces
   `adapterId == keccak256(adapter_program.key) mod p` on chain.
   In-circuit it is a passthrough, so a proof for adapter X cannot
   verify for adapter Y (different public input → verification fails).

### In-circuit verification of the adapter binding

Direct quote from `adapt.circom`:

```
component actionHasher = Poseidon(3);
actionHasher.inputs[0] <== adaptBindTag;
actionHasher.inputs[1] <== actionPayloadKeccakFr;
actionHasher.inputs[2] <== expectedOutMint;
actionHash === actionHasher.out;
```

Pool-side recomputation (from
[`programs/b402-pool/src/instructions/adapt_execute.rs`](https://github.com/b402-ai/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs)):

```rust
let payload_keccak = keccak::hash(&args.action_payload).0;
let payload_keccak_fr = util::reduce_le_mod_p(&payload_keccak);
let expected_out_mint_fr = util::reduce_le_mod_p(&pi.expected_out_mint.to_bytes());
let expected_action_hash = hashv(
    Parameters::Bn254X5,
    Endianness::LittleEndian,
    &[&TAG_ADAPT_BIND[..], &payload_keccak_fr[..], &expected_out_mint_fr[..]],
)
.map_err(|_| error!(PoolError::InvalidInstructionData))?
.to_bytes();
require!(
    pi.action_hash == expected_action_hash,
    PoolError::InvalidAdapterBinding
);
```

The `Endianness::LittleEndian` setting is load-bearing: the on-chain
`sol_poseidon` syscall uses little-endian, matching the circuit's field
encoding.

## Sub-circuits

### `Commitment` template (`circuits/lib/commitment.circom`)

```
out = Poseidon_5(commitTag, tokenMint, value, random, spendingPub)
```

One Poseidon_5 hash, ~280 R1CS constraints.

### `Nullifier` template (`circuits/lib/nullifier.circom`)

```
out = Poseidon_3(nullTag, spendingPriv, leafIndex)
```

One Poseidon_3 hash, ~240 R1CS.

### `SpendingPub` template (`circuits/lib/spending_pub.circom`)

```
out = Poseidon_2(spendKeyPubTag, spendingPriv)
```

### `MerkleRoot` template (`circuits/lib/merkle_verify.circom`)

26 levels of Poseidon_3 with `pathBits` selectors. ~6,240 R1CS per
proven input.

### `RangeCheck(N)` template

Generic N-bit decomposition + sum-equals-original check. Used at
N=64 for amounts, N=128 for owner pubkey halves.

## Trusted setup status

| Circuit | Phase 2 status | VK location |
|---|---|---|
| `transact` | Throwaway single-contributor | `circuits/build/ceremony/transact_verification_key.json` |
| `adapt` | Throwaway single-contributor | `circuits/build/ceremony/adapt_verification_key.json` |

Both VKs are baked into their respective verifier programs at build time
via `scripts/vk-to-rust.mjs`. Mainnet requires re-running the ceremony
with ≥3 contributors per [`PRD-08`](https://github.com/b402-ai/b402-solana/blob/main/docs/prds/PRD-08-audit-launch.md).

## Performance

From the assurance benchmark report
([`reports/benchmark-summary.md`](https://github.com/b402-ai/b402-solana-assurance/blob/main/reports/benchmark-summary.md)):

| Phase | Avg ms |
|---|---|
| Wallet build | 0.74 |
| Merkle append (TS) | 3.80 |
| Note encrypt + decrypt | 3.06 |
| **Proof generation (snarkjs WASM)** | **733.36** |
| Verifier CLI (Rust) | 116.79 |
| Adapt fixture load | 0.09 |
| Note scan + decrypt | 3.09 |

Proof generation dominates wall time. Rapidsnark (C++) would cut this to
~250 ms; v1 stays on snarkjs for portability and zero-install browser
compatibility.
