`adapt_execute` is the SDK primitive for "spend a shielded note + run a
DeFi action + reshield the output." This page covers the
TypeScript surface — the prover, the action-payload encoding, and the
end-to-end flow.

## `AdaptProver`

```typescript
import { AdaptProver } from '@b402ai/solana-prover';

const prover = await AdaptProver.create({
  wasmPath: '/circuits/build/adapt_js/adapt.wasm',
  zkeyPath: '/circuits/build/ceremony/adapt_final.zkey',
});

const proof = await prover.prove({
  // Public inputs (23 total — see /protocol/circuits)
  merkleRoot,
  nullifier: [n0, n1],
  commitmentOut: [c0, c1],
  publicAmountIn,
  publicAmountOut: 0n,        // adapt forces this to 0
  publicTokenMint,            // IN mint as Fr
  relayerFee,
  relayerFeeBind,
  rootBind,
  recipientBind,
  adapterId,                  // keccak256(adapter_program_id) reduced mod p
  actionHash,                 // see below
  expectedOutValue,
  expectedOutMint,            // OUT mint as Fr

  // Private inputs
  inTokenMint, inValue, inRandom, inSpendingPriv, inLeafIndex,
  inSiblings, inPathBits, inIsDummy,
  outValue, outRandom, outSpendingPub, outIsDummy,
  relayerFeeRecipient,
  recipientOwnerLow, recipientOwnerHigh,
  actionPayloadKeccakFr,      // keccak256(action_payload) reduced mod p
});
```

Output:

```typescript
{ proof: Uint8Array(256), publicInputs: Uint8Array[] /* 23 elements */ }
```

Wall time: ~800 ms in snarkjs WASM on a modern CPU. Rapidsnark would
be ~250 ms; not yet wired in v1.

## Computing `actionHash`

The single most important client-side calculation. Wrong `actionHash`
= circuit constraint fails = unrecoverable proof.

```typescript
import { keccak256 } from 'ethers'; // or any keccak impl
import { poseidon3 } from '@b402ai/solana';

// 1. Build action_payload (Borsh-encoded variant for the adapter).
const actionPayload = serializeJupiterRoute(jupRouteIxData);
//                  = serializeKaminoAction({Deposit: {...}});
//                  = ...

// 2. keccak the bytes, reduce mod BN254 scalar field prime.
const payloadKeccak = keccak256(actionPayload);            // 32 bytes
const payloadKeccakFr = reduceLeMmodP(payloadKeccak);      // Fr

// 3. expectedOutMint as Fr.
const expectedOutMintFr = reduceLeMmodP(expectedOutMint.toBytes()); // Fr

// 4. Poseidon-3 combine.
const actionHash = poseidon3(
  TAG_ADAPT_BIND,        // "b402/v1/adapt-bind" embedded as Fr
  payloadKeccakFr,
  expectedOutMintFr,
);
```

The on-chain handler in
[`programs/b402-pool/src/instructions/adapt_execute.rs`](https://github.com/b402-ai/b402-solana/blob/main/programs/b402-pool/src/instructions/adapt_execute.rs)
recomputes this exactly:

```rust
let payload_keccak = keccak::hash(&args.action_payload).0;
let payload_keccak_fr = util::reduce_le_mod_p(&payload_keccak);
let expected_out_mint_fr = util::reduce_le_mod_p(&pi.expected_out_mint.to_bytes());
let expected_action_hash = hashv(
    Parameters::Bn254X5,
    Endianness::LittleEndian,
    &[&TAG_ADAPT_BIND[..], &payload_keccak_fr[..], &expected_out_mint_fr[..]],
)?
.to_bytes();
require!(pi.action_hash == expected_action_hash, ...);
```

`Endianness::LittleEndian` is load-bearing — both the on-chain
`sol_poseidon` syscall and the snarkjs Poseidon use little-endian Fr
encoding.

## End-to-end Jupiter swap (verified flow)

From [`examples/swap-e2e-jupiter.ts`](https://github.com/b402-ai/b402-solana/blob/main/examples/swap-e2e-jupiter.ts):

```typescript
// 1. Fetch a real Jupiter route quote.
const quote = await jupiterClient.quoteGet({
  inputMint: WSOL,
  outputMint: USDC,
  amount: 100_000_000n,        // 0.1 wSOL
  slippageBps: 50,
});

// 2. Get the route ix.
const swapIx = await jupiterClient.swapInstructionsPost({
  quoteResponse: quote,
  userPublicKey: jupiterAdapterPda.toBase58(),  // adapter authority
});

// 3. action_payload = swap.swapInstruction.data.
const actionPayload = Buffer.from(swapIx.swapInstruction.data, 'base64');

// 4. Generate the adapt proof.
const proof = await adaptProver.prove({
  // ... all public + private inputs
  actionHash: poseidon3(TAG_ADAPT_BIND, keccak256(actionPayload), USDC_Fr),
  expectedOutValue: parseQuoteMinOut(quote),
  expectedOutMint: USDC_Fr,
  actionPayloadKeccakFr: reduceLeMmodP(keccak256(actionPayload)),
});

// 5. Build the adapt_execute ix.
const ix = await buildAdaptExecuteIx({
  pool,
  adapterProgram: jupiterAdapterId,
  proof: proof.proof,
  publicInputs: proof.publicInputs,
  rawAdapterIxData: Buffer.concat([
    EXECUTE_DISCRIMINATOR,    // first 8 bytes
    swapIx.swapInstruction.data,  // forward Jupiter's data
  ]),
  actionPayload,
  remainingAccounts: swapIx.swapInstruction.keys,
});

// 6. Submit (with the b402 ALT for tx size).
const tx = new VersionedTransaction(
  new TransactionMessage({
    instructions: [computeBudgetIx, ix],
    recentBlockhash: blockhash,
    payerKey: relayer.publicKey,
  }).compileToV0Message([b402Alt])
);

await connection.sendTransaction(tx);
```

## `MAX_ACTION_PAYLOAD`

Exported from `@b402ai/solana`:

```typescript
export const MAX_ACTION_PAYLOAD = 350; // bytes
```

This is a soft cap to keep the tx under the 1,232 B Solana limit. The
actual on-chain handler does not enforce a payload length cap — the
limit is the tx-size cap. PRD-04 §5.3 documents the 400 B figure;
the SDK uses 350 to leave headroom.

## Self-submission vs relayer

Same as shield/unshield — pass `relayer` as the user's keypair to
self-submit, or use a `RelayerHttpClient` to delegate.
